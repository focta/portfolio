# 概要

自己紹介に利用するポートフォリオサイトです。
ローカルに構築いただいて確認することもできます！

## アプリの起動

本プロジェクトの開発時には Bun を使用したため、Bun によるアプリの起動が可能です。

### 必要な環境

- Bun (最新版推奨)
- Node.js 18 以上（Bun の動作要件）

### 起動方法

```bash
# 依存関係のインストール
bun install

# 開発サーバーの起動
bun dev
```

ローカルでの起動後は[http://localhost:3000](http://localhost:3000)をブラウザにてご確認ください。

### その他のコマンド

```bash
# プロダクションビルド
bun run build

# プロダクションサーバーの起動
bun run start

# リントの実行
bun run lint
```

# アプリ概要説明

本ポートフォリオサイトは、1 ページスクロール型の構成で以下のセクションを実装予定です。

## 実装済みセクション

### Hero セクション

- PC/SP の両方でのファーストビューを構成している部分
- 名前、タイトル、自己紹介文、SNS リンク（GitHub、Twitter、Email）を表示
- レスポンシブデザインとダークモードに対応

### About Me セクション

- 詳細な自己紹介文を表示
- 興味・関心事をリスト形式で表示
- 大切にしていること（価値観）をバッジ形式で表示
- レスポンシブデザインとダークモードに対応

### Skills セクション

- 技術スタックをカテゴリ別（フロントエンド、バックエンド、ツール、その他）にグループ化して表示
- 各スキルのレベルを星（★）で表示
- レスポンシブデザインとダークモードに対応

### Career セクション

- キャリア履歴と開発案件をタイムライン形式で表示
- 役職、会社名、期間、業務内容、成果を表示
- 開発案件を折りたたみ可能な形式で表示（技術スタック、期間、リンクを含む）
- 古い順（時系列順）で表示
- 期間を日本語形式で表示（例: "2020年4月 - 2023年3月"）
- レスポンシブデザイン（モバイルはシンプルなリスト、デスクトップはタイムライン形式）
- ダークモードに対応

## コンテンツ設計方針

各セクションの役割を明確にし、重複を避けながら情報を整理しています。

### Hero セクションと About Me セクションの役割分担

| セクション   | 役割                     | 内容の特徴                 |
| ------------ | ------------------------ | -------------------------- |
| **Hero**     | 第一印象・キャッチコピー | 簡潔・印象的・専門性の提示 |
| **About Me** | 詳細な自己紹介           | 具体的・経験・ストーリー   |

- **Hero セクション**: ファーストビューとして、簡潔で印象的な自己紹介文を表示します。訪問者が最初に目にする部分のため、専門性や興味を端的に伝えることを重視しています。
- **About Me セクション**: Hero セクションで興味を持った訪問者に対して、より詳細で具体的な自己紹介を提供します。経験年数、実績、技術的な専門性、キャリアのストーリーなどを含め、改行を活用して読みやすく整理しています。

# 技術スタック

## フレームワーク・ライブラリ

- **Next.js 16.0.3** - React フレームワーク（App Router 使用）
- **React 19.2.0** - UI ライブラリ
- **TypeScript 5** - 型安全性のための言語

## スタイリング

- **Tailwind CSS v4** - ユーティリティファーストの CSS フレームワーク
- ダークモード対応（`prefers-color-scheme`を使用）

## 開発環境・ツール

- **Bun** - パッケージマネージャー・ランタイム
- **ESLint** - コード品質チェック
- **Next.js ESLint Config** - Next.js 用の ESLint 設定

## アーキテクチャ

- **App Router** - Next.js の最新ルーティングシステム
- **コンポーネントベース設計** - 再利用可能なコンポーネント構成
- **型定義の分離** - `types/index.ts`で型定義を一元管理
- **データの分離** - JSON ファイルでデータを管理（将来的に Supabase/API への移行を想定）

## パスエイリアス

- `@/*` - プロジェクトルートを参照するパスエイリアスを設定

## 特徴

- **型安全性**: TypeScript による厳密な型チェック
- **パフォーマンス**: Next.js の自動最適化機能を活用
- **レスポンシブ**: モバイル・タブレット・デスクトップに対応
- **アクセシビリティ**: 適切な aria-label などの実装
- **メンテナンス性**: コンポーネントとデータの分離による保守性の向上
